# riji-agent 产品需求文档（PRD）

**状态**：v0.1 / 2026-06-23  
**产品定位**：一个 local-first 的 Markdown 日记 Agent 网关：日记 vault、索引、草稿和审计留在用户本机，外部 Agent / 模型只能通过受限工具访问最小必要片段。默认栈为 Feishu + Hermes + DeepSeek，但架构上允许替换 IM、Agent runtime 和模型 Provider。

## 1. 背景与目标

用户在 Obsidian 或类似 Markdown vault 中持续记录日记、周记和月报，希望以自然对话的方式回顾、提问、梳理个人经历，并可在不同对话中使用不同人设。项目应能服务单个用户的本地私密部署，也应能作为开源组件被其他人接入自己的 IM、Agent runtime 和模型供应商。

产品的核心目标：

1. 使用云端旗舰模型获得高质量的复杂推理、模糊语义理解与长时间线归纳能力。
2. 日记原始 Markdown、索引、对话历史和访问控制始终由用户自己的设备保存和控制。
3. 模型通过受限的本地只读工具，进行多轮、带时间意识的检索，而非只接收一次固定的 RAG 片段。
4. 第一阶段提供 Feishu + Hermes + DeepSeek 的默认开箱栈，用于与不同导师对话和记录日记；同时把 Feishu、Hermes、DeepSeek 视为可替换模块。
5. 基于用户反馈逐步改进检索和人设，但任何长期记忆或人格变更必须先由用户确认。

## 2. 已确认的产品决策

| 项目 | 决策 |
| --- | --- |
| 主模型 | 默认使用 DeepSeek 云端 API 与 `deepseek-reasoner`；日常快速问答可按配置切换到 `deepseek-chat`。模型层通过 Provider Adapter 隔离，支持后续替换为其他 OpenAI-compatible 或专用 provider。 |
| 本地模型 | 不使用 Ollama 运行聊天/推理模型。可选的小型本地 embedding 模型只用于语义索引，不参与生成回答。 |
| 日记位置 | 源日记库由 `RIJI_JOURNAL_ROOT` 配置，指向用户自己的 Markdown journal 目录。项目不复制该目录。 |
| 项目位置 | 可安装为本地 Python package，也可从仓库 checkout 运行；运行数据默认位于 `RIJI_DATA_DIR`。 |
| 对话入口 | 默认入口为 Feishu 私聊；IM 层应作为 adapter，后续可替换为其他聊天入口或本地 CLI/Web。 |
| Agent 编排 | 默认使用 Hermes 负责 IM 会话路由、导师选择、定时任务与技能；它只能通过 riji-agent 的受限工具访问日记。 |
| 默认权限 | 只读日记。任何写入日记、修改标签或保存长期记忆，都必须展示变更并经用户确认。 |

## 3. 非目标

- 不追求完全离线推理。
- 不将整个 vault、云端文件上传、云端向量库作为默认方案。
- 不允许模型拥有任意文件系统、终端、网络或自动写日记权限。
- 不让 Agent 在无人审核下自动改变人设、长期记忆或日记原文。
- MVP 不以独立 Web/PWA 作为入口；该能力留给后续阶段。

## 4. 隐私边界

这是“**本地数据控制 + 云端推理**”方案，**not a zero-egress system**，也不是“文字完全不离开设备”的方案。

本地组件负责读取、切分、建立索引和筛选证据。云端模型只会收到：用户问题、当前人设指令、以及本地工具返回的 bounded journal snippets。complete vault、原始 Markdown 文件、SQLite 数据库和 API keys 不会由 riji-agent 上传。

必须实现：

1. 禁用云端文件上传、云端向量库和云端持久化会话。
2. 每次工具调用记录实际发送的片段、来源文件和时间，便于审计。
3. API Key 仅存于本地网关的环境变量，绝不下发给浏览器或手机端。
4. 本地服务仅监听 localhost 或 Tailscale 网络，不开放公网端口。

带有 `private: true` frontmatter 的日记不得作为检索结果或 `read_note` 内容出云。系统还必须遵守最小片段出云、来源审计与不上传完整 vault 的规则。

## 5. 总体架构

```text
Feishu 私聊 / 指令 / @导师（默认 IM adapter）
        ↕  Hermes 内置 Feishu 接入（默认 agent runtime）
Hermes（路由、导师会话、定时任务、技能）
        ↕  受限工具调用
riji-agent 本地网关（人设、权限、审计、共享记忆）
        ↕
本地索引服务（日期/标题/标签/全文/语义向量） ← Obsidian riji Markdown（只读）
        ↕  HTTPS API；仅发送必要片段
DeepSeek 云端模型（默认 model provider）
```

### 5.1 为什么采用 Agentic RAG

传统 RAG 先按固定规则取 Top-K 片段，再让模型回答；它很容易遗漏隐喻、情绪表达和时间转折。

本产品由旗舰模型决定下一步检索策略，但实际搜索仍由本地工具执行。例如用户问“去年下半年为什么工作没有意义”，模型可以先限定 7–12 月，搜索“意义感/疲惫/组织变化”，发现 10 月是转折后，再读取前后数周的日记并形成有证据的结论。

## 6. 本地工具设计（MVP）

所有工具均由本地网关实现、仅可读取经过允许的日记范围。

| 工具 | 参数示例 | 用途 |
| --- | --- | --- |
| `search_journal` | `query`, `date_from`, `date_to`, `tags`, `top_k` | 关键词和语义混合检索，返回片段、日期、标题、来源。 |
| `timeline` | `topic`, `date_from`, `date_to`, `granularity` | 按日/周/月返回主题出现、事件与情绪线索。 |
| `read_note` | `path` 或 `date` | 在检索命中后读取单篇允许的日记。 |
| `find_before_after` | `date`, `days`, `topic` | 分析某一转折点前后的相关记录。 |
| `list_periods` | `kind`, `date_from`, `date_to` | 返回可用日记、周记、月报的时间范围与缺失情况。 |
| `draft_daily_entry` | `content`, `date`, `persona_id` | 从飞书消息生成日记草稿，不写入文件。 |
| `commit_draft` | `draft_id`, `user_confirmation` | 仅在用户明确确认后写入日记并触发增量索引。 |

工具返回必须包括稳定来源 ID，如 `[[riji/daily/2026-06-18]]`。最终回答必须引用实际使用的来源，并区分“日记事实”“模型推断”和“不足以判断”。

## 7. 人设与对话

一个人设是一个独立配置，而不是一次临时 prompt。其配置至少包含：

- 名称、头像、描述和系统提示词；
- 可检索的日记范围与允许使用的工具；
- 语气和回答结构；
- 是否允许追问、挑战用户叙事、给出行动建议；
- 独立会话历史与经确认的长期记忆。

首批建议人设：

1. **温柔回顾者**：注重理解、复述和情绪容纳，不轻易下结论。
2. **直率教练**：指出叙事中的回避、矛盾和可执行选择，保持尊重。
3. **未来的我**：以指定未来视角反思，但不能伪造日记中不存在的事实。
4. **王阳明导师**：以王阳明思想为主要框架，围绕“心即理、知行合一、致良知”等概念，帮助用户从内在动机、真实认知和具体行动之间的关系反思问题。

### 7.1 王阳明导师的知识与表达边界

- 建立独立、可追溯的王阳明思想资料库，优先收录可靠版本的《传习录》及必要的研究/注释资料；资料与用户日记库分开管理。
- 回答时同时可检索日记证据与思想资料：日记用于理解用户实际处境，思想资料用于形成反思框架。
- 使用“受王阳明思想启发的导师口吻”，不得声称自己是真实的王阳明，也不得编造原文、经历、书信或历史判断。
- 涉及具体思想、原文或观点归属时，标明资料来源；无法确认出处时，以概括性表述并说明是现代阐释。
- 不将历史思想包装为医学、心理治疗、法律或投资结论；遇到高风险困扰时，保持支持性并建议寻求合适的专业帮助。

### 7.2 共享记忆与导师隔离

所有导师共享同一份日记库、已确认的长期记忆和用户偏好；每个导师隔离保存其系统提示词、会话历史和未确认的工作笔记。不得将一个导师的私有会话内容自动注入另一个导师的上下文。

## 8. Hermes 与飞书（MVP 入口）

Hermes 是默认 Agent runtime，使用其自带的 Feishu 接入作为默认 IM 路径，不另行开发 Feishu Bot 协议适配器。具体配置以安装时 Hermes 版本提供的官方 Feishu 接入说明为准。后续架构应允许替换 IM adapter、Agent runtime 和模型 Provider。

### 8.1 路由规则

1. 用户在飞书私聊中通过 `@温柔回顾者`、`@直率教练`、`@未来的我`、`@王阳明导师` 或导师选择命令进入对应人设。
2. Hermes 将 `persona_id`、飞书用户 ID 和会话 ID 传给 riji-agent；riji-agent 决定可用记忆、日记权限和工具范围。
3. 同一飞书用户的不同导师会话独立保存；共享记忆仅由 riji-agent 统一读取。
4. 定时复盘、提醒和候选记忆生成由 Hermes 的 cron/skill 执行，但写入仍需 riji-agent 的确认流程。

### 8.2 飞书日记写入流程

1. 用户发送“记录日记：……”或进入日记记录指令。
2. 导师调用 `draft_daily_entry`，在飞书中返回日期、标题、正文和标签的草稿预览。
3. 用户明确回复“确认保存”后，导师调用 `commit_draft`。
4. riji-agent 写入目标 Markdown 文件、记录审计事件并更新本地索引。
5. 系统返回写入的 Obsidian wikilink；非明确确认、重复事件或群聊消息均不得写入。

### 8.3 飞书安全边界

- 仅允许白名单飞书用户与私聊会话访问日记工具；群聊默认不具备检索和写入权限。
- 校验 Hermes/飞书连接提供的身份与事件签名；对消息事件做幂等处理，防止重复写入。
- 飞书消息与机器人回复会经过飞书服务；因此本方案不满足“任何文字绝不出设备”。
- Hermes 不获得日记目录的直接读写权限；只允许调用 riji-agent 的窄工具接口。

## 9. 自我改进机制

“自我进化”定义为受控、可审计的改善，而不是无授权的自我修改。

1. 用户可以标记回答为“有用”“不准确”并写下修正。
2. 系统定期生成三类**候选项**：长期记忆、检索规则改进、人设提示词改进。
3. 用户查看证据与 diff 后确认；只有确认的候选项写入本地配置或记忆库。
4. Hermes 的定时任务和技能可生成候选项；候选项仍必须由用户审核后才可生效。

## 10. 用户故事与验收标准

### US-01：带来源的日记问答

作为用户，我想询问过往经历并查看依据，避免 AI 编造。

**验收**：回答中列出使用过的 `[[riji/...]]` 来源；没有来源时明确说“日记中未找到足够证据”。

### US-02：复杂时间性回顾

作为用户，我想问“去年下半年”“某件事发生前后”的变化。

**验收**：模型会调用时间范围工具，必要时多轮检索；回答注明时间窗口与转折点依据。

### US-03：多个人设

作为用户，我想在不同对话中选择不同人设，且它们互不混淆记忆。

**验收**：人设拥有独立提示词和会话/长期记忆空间；切换人设后不泄露另一人设的私有记忆。

### US-04：飞书中的导师对话与日记记录

作为用户，我想在飞书中和指定导师聊天，并通过同一入口写日记。

**验收**：飞书私聊可稳定路由到指定导师；日记写入必须经过按模板区块生成的 patch 预览和“确认保存”，且写入结果可回链到 Obsidian。

## 11. MVP 实施范围

### 阶段 1：可用的私密问答

- Markdown 日记增量索引；
- 日期、标签、全文检索与本地语义检索；
- Hermes + 飞书内置接入；
- 多导师人设、单飞书用户、共享已确认记忆与隔离会话；
- 王阳明导师及其独立、可引用的思想资料库；
- 旗舰模型的工具调用循环；
- 飞书日记草稿、按模板区块定向追加、确认写入、引用来源与审计日志。

### 阶段 2：日记 Agent 体验

- 多人设与独立记忆；
- Timeline/前后对比工具；
- 独立 Web/PWA + Tailscale（作为飞书之外的可选入口）；
- 反馈与候选记忆审核界面。

### 阶段 3：受控的主动性

- 周/月复盘草稿；
- 用户批准的日记写回；
- 扩展 Hermes 的定时任务与技能候选生成。

## 12. 初步技术栈

| 层级 | 选择 |
| --- | --- |
| 本地 Agent 网关 | Python + FastAPI |
| 日记解析 | Python Markdown/frontmatter 解析器 |
| 检索 | SQLite FTS5 + sqlite-vec（或本地 Qdrant） |
| 语义 embedding | 小型本地 embedding 模型；可关闭并退化为全文检索 |
| 云端推理 | DeepSeek API（OpenAI 兼容接口）；通过 Provider Adapter 隔离供应商与模型名，支持未来替换 |
| Agent/渠道编排 | 默认 Hermes + 内置 Feishu 接入；模块化后可替换 |
| 前端 / IM | 默认 Feishu；后续可增加 CLI、独立 Web/PWA 或其他 IM adapter |
| 私网访问 | 供本地开发和后续 Web/PWA 使用的 Tailscale |
| 本地持久化 | SQLite（会话、配置、审计日志、确认后的记忆） |

## 13. 开发前待确认项

1. 是否只接入 DeepSeek，或在后续加入 Anthropic / Gemini 等备用模型提供商。
2. 飞书租户、Bot 权限与 Hermes 内置飞书接入所需的配置是否已具备。
3. 电脑是否可长期运行，以决定 Hermes、后台增量索引与定时任务策略。
